RAG — Retrieval Augmented Generation

LLM ne odgovara „iz glave" — prvo dohvati relevantne dijelove vaših dokumenata, pa tek onda generiše odgovor.

Analogija: referent otvori fascikl, označi stranice, pa pozove eksperta da objasni samo na osnovu označenog.

Svrha RAG-a u instituciji

1. Nema vaših podataka

LLM je treniran na javnom internetu do određenog datuma — ne zna interne pravilnike.

2. Context window

Cijeli arhiv ne staje u prompt i košta previše tokena.

3. Halucinacije

Bez izvora model izmišlja činjenice. RAG smanjuje rizik uz dobar kontekst.

4. Traceability

U državnoj upravi trebate znati odakle dolazi informacija — citati obavezni.

Kada šta koristiti?

Pristup Kada Prednost Mana
Dugi prompt < 20 strana teksta Nema infrastrukture Ne skalira, skupi tokeni
RAG Veliki korpus, često ažuriranje Svježi podaci, citati Retrieval mora biti dobar
Fine-tuning Stil, format, domena Brži odgovori Skupo, loše za nove činjenice
Function calling Live API podaci Tačni brojevi sada Ne za statički PDF arhiv

RAG samo mijenja šta stavite u prompt — ne mijenja težine modela. Za propise koji se mijenjaju kvartalno, RAG je praktičniji od fine-tuninga.

Jednostavan primjer kako možemo predstaviti riječi kao brojeve u tri dimenzije:

Kada ove brojeve smjestimo u prostor, "Kralj" i "Kraljica" će stajati jako blizu jedan drugog, dok će "Jabuka" biti u potpuno drugom ćošku sobe.

Kako se to koristi u RAG-u? (Semantička pretraga)

Kada korisnik postavi pitanje: "Gdje mogu predati zahtjev za ličnu kartu?", RAG sistem radi sljedeće:

  1. Pitanje se šalje embedding modelu koji od njega napravi vektor pitanja (niz brojeva).
  2. Sistem pretražuje vektorsku bazu tražeći tekstove (chunkove) čiji su vektori najbliži.
  3. Baza pronalazi: "Izdavanje identifikacionih dokumenata vrši se u prostorijama MUP-a".

Obratite pažnju

Riječi u pitanju i odgovoru se uopšte ne poklapaju (zahtjev vs. izdavanje, lična karta vs. identifikacioni dokumenti), ali zahvaljujući embeddingu, sistem zna da imaju isto značenje.

Jednostavna analogija:

Zamislite da svaka riječ na svijetu dobije svoju GPS koordinatu. Umjesto na mapi svijeta, te koordinate se nalaze na "mapi ljudskog znanja". Embedding model svakoj rečenici dodijeli njenu GPS lokaciju na osnovu njenog smisla. RAG sistem onda samo pogleda koje su rečenice na mapi najbliže komšije korisnikovom pitanju.

Halucinacije — i kako ih smanjiti

Bez RAG-a

„Koliko turista posjeti Sutjesku?" → Model izmisli broj npr. 50.000

S RAG-om

Chunk sa Sutjeskom bez broja → „U dokumentu nemam tu informaciju."

5 tehnika za smanjenje halucinacija

  1. Strogi system prompt („SAMO iz konteksta")
  2. Niska temperatura (0.1–0.3)
  3. Top-k samo visokokvalitetni chunkovi
  4. Odbijanje odgovora kad nema retrieval rezultata
  5. Ljudska revizija za kritične odluke

RAG nije magija — kvaliteta odgovora = kvaliteta retrievala + kvaliteta prompta + disciplina modela. Zato dan6-rag prikazuje bodove i odlomke u UI-ju.

Kod — korak 1

dokument.py — učitavanje i chunking

from pathlib import Path DOKUMENT_PATH = ( Path(__file__).resolve() .parent.parent.parent / "podaci" / "bih_turizam.txt" ) def ucitaj_dokument(putanja=DOKUMENT_PATH): if not putanja.exists(): raise FileNotFoundError(...) return putanja.read_text(encoding="utf-8") def podijeli_na_dijelove(tekst): dijelovi = [] for blok in tekst.split("\n\n"): blok = blok.strip() if not blok: continue if blok.startswith("NAPOMENA:"): continue if len(blok) < 30: continue dijelovi.append(blok) return dijelovi
  1. Path resolving — 3× parent: dan6-rag → projekti → repo root
  2. UTF-8 — obavezan za čćžšđ
  3. Split \n\n — paragrafi kao chunkovi
  4. Filter NAPOMENA — meta-tekst ne ide u RAG
  5. Min 30 char — prekratki fragmenti su šum
python -c "from dokument import DIJELOVI; \ print(len(DIJELOVI), 'odlomaka')" → ~15-20 odlomaka
Kod — korak 2

pretraga.py — keyword retrieval

STOP_RIJECI = {"a", "i", "je", "su", "za", "šta", ...} def ocisti_rijeci(tekst): rijeci = re.findall(r"[a-zA-ZčćžšđČĆŽŠĐ]+", tekst.lower()) return {r for r in rijeci if r not in STOP_RIJECI and len(r) > 2} def izracunaj_bod(pitanje, odlomak): return len(ocisti_rijeci(pitanje) & ocisti_rijeci(odlomak)) def pronadi_relevantne(pitanje, dijelovi, top_k=2): rezultati = [] for odlomak in dijelovi: bod = izracunaj_bod(pitanje, odlomak) if bod > 0: rezultati.append({"tekst": odlomak, "bod": bod}) rezultati.sort(key=lambda x: x["bod"], reverse=True) return rezultati[:top_k]

Kako radi?

  1. Očisti tekst → set riječi (bez stop-riječi)
  2. Presjek skupova pitanja ∩ odlomka = bod
  3. Sortiraj opadajuće → uzmi top_k
Pitanje Top-1 Bod
Sutjeska Perućica NP Sutjeska odlomak 2
UNESCO most Stari Most odlomak 2
Pariz stanovnika (prazno) 0
