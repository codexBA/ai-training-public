🗂️ Virtualno okruženje (venv)

Svaki Python projekat bi trebao imati vlastito virtuelno okruženje — izolirani folder koji sadrži samo biblioteke potrebne za taj projekat. Bez njega, sve instalacije idu globalno na sistem i nastaje "dependency hell".

📋 Kompletni workflow — od nule do pokretanja

# Korak 1: Idite u folder projekta
cd projekti\dan1-tokeni

# Korak 2: Kreirajte venv (samo jednom po projektu)
python -m venv venv

# Korak 3: Aktivirajte venv (svaki put kad otvorite terminal)
.\venv\Scripts\Activate.ps1

# Korak 4: Instalirajte biblioteke iz requirements.txt (samo jednom)
pip install -r requirements.txt

# Korak 5: Pokrenite server
python -m uvicorn main:app --reload --port 8001
🔁 Svaki sljedeći put (venv već postoji): Samo aktivirajte i pokrenite — koraci 1, 3 i 5. Nema potrebe ponavljati kreiranje ni instalaciju.

Brisanje i reset venv-a

Ako želite krenuti ispočetka ili instalacija nije uspjela:

# Deaktivirajte trenutni venv (ako je aktivan)
deactivate

# Obrišite venv folder (Windows PowerShell)
Remove-Item -Recurse -Force .\venv

# Kreirajte novi i ponovo instalirajte
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
⚠️ venv ne ide na GitHub! Folder venv/ sadrži stotine MB koda. Nikada ga ne commitujte — on je u .gitignore. Šaljite samo requirements.txt.

📦 pip — upravljač paketima

pip je alat koji instalira Python biblioteke sa interneta (PyPI). Uvijek koristite pip unutar aktiviranog venv-a.

KomandaŠta radi
pip install paketInstalira jedan paket
pip install -r requirements.txtInstalira sve pakete sa liste
pip listPrikazuje sve instalirane pakete
pip show paketDetalji o jednom paketu (verzija, lokacija)
pip uninstall paketUklanja paket
pip freeze > requirements.txtIzvozi sve instalirane pakete u fajl
python -m pip install --upgrade pipAžurira pip na najnoviju verziju

requirements.txt — lista zavisnosti

Ovo je tekstualni fajl koji popisuje sve pakete i njihove verzije. Možete koristiti == za tačnu verziju ili >= za minimalnu: 

# Tačna verzija — garantuje reproduktibilnost
fastapi==0.115.0
uvicorn==0.30.0

# Minimalna verzija — dozvoljava novije verzije
openai>=1.50.0
tiktoken>=0.8.0   # >=0.8.0 jer 0.7.0 ne radi na Python 3.13
⚠️ Windows + naša slova u komentarima: Ako komentari u requirements.txt sadrže č, ć, š, đ, ž — pip može javiti UnicodeDecodeError. Rješenje: koristite samo ASCII znakove u komentarima (zamijenite č→c, ć→c, itd.)

🔌 API pozivi prema LLM-ovima

Svi projekti koriste OpenAI SDK za komunikaciju s AI modelima. Isti kod radi i za DeepSeek (cloud) i za Ollama (lokalno) — razlikuju se samo api_key i base_url.

Minimalni primjer — sinhroni poziv

from openai import OpenAI
import os
from dotenv import load_dotenv

load_dotenv()  # Ucitava .env fajl

# Kreiranje klijenta
klijent = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com"
)

# Slanje zahtjeva
odgovor = klijent.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "Odgovaraj kratko na bosanskom."},
        {"role": "user",   "content": "Sta je tokenizacija?"}
    ],
    temperature=0.7,
    max_tokens=200
)

# Citanje odgovora
tekst = odgovor.choices[0].message.content
print(tekst)

Asinhroni poziv (koristi se u FastAPI)

from openai import AsyncOpenAI
import asyncio

klijent = AsyncOpenAI(
    api_key="your-key",
    base_url="https://api.deepseek.com"
)

async def pitaj_ai(prompt: str) -> str:
    odgovor = await klijent.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.7
    )
    return odgovor.choices[0].message.content

# Pokretanje async funkcije
tekst = asyncio.run(pitaj_ai("Objasni tokenizaciju"))

Struktura messages[] polja

Svaki API poziv sadrži polje messages — listu poruka u konverzaciji. Svaka poruka ima role i content:

RoleKo je to?Primjena
system Instrukcija za AI — njegova "ličnost" i pravila Definisanje tona, jezika, formata odgovora
user Pitanje ili tekst koji korisnik šalje Konkretan zadatak ili pitanje
assistant Prethodni AI odgovor (za multi-turn chat) Historija razgovora u chat aplikacijama

Ključni parametri API poziva

ParametarTipOpisPreporučena vrijednost
model string Ime modela koji se koristi deepseek-chat, llama3.2:1b
temperature float 0.0–2.0 Kreativnost: 0.0=deterministički, 1.5=haotičan 0.0 (fakti), 0.7 (chat), 1.0 (kreativno)
max_tokens int Maksimalna dužina odgovora u tokenima 150–500 za kratke odgovore
messages list[dict] Historija konverzacije Uvijek uključite system prompt
stream bool Streaming odgovor token po token False za API, True za chat UI

Fallback: DeepSeek → Ollama

Ovaj obrazac se ponavlja u svakom projektu — automatski prelazi na lokalni model ako nema API ključa:

def get_llm_client():
    """Vraca klijent i model. Prioritet: DeepSeek -> Ollama."""
    import httpx

    deepseek_key = os.getenv("DEEPSEEK_API_KEY", "")

    # Pokusaj 1: DeepSeek cloud API
    if deepseek_key and deepseek_key.startswith("sk-"):
        return AsyncOpenAI(
            api_key=deepseek_key,
            base_url="https://api.deepseek.com"
        ), "deepseek-chat"

    # Pokusaj 2: Ollama lokalno
    ollama_url = os.getenv("OLLAMA_BASE_URL", "http://localhost:11434")
    try:
        httpx.get(f"{ollama_url}/api/tags", timeout=2.0)
        model = os.getenv("OLLAMA_MODEL_SMALL", "llama3.2:1b")
        return AsyncOpenAI(api_key="ollama", base_url=f"{ollama_url}/v1"), model
    except:
        raise RuntimeError("Ni DeepSeek ni Ollama nisu dostupni!")

⚡ Async/Await — paralelni API pozivi

Asinhrone funkcije omogućavaju da čekate na više API poziva istovremeno umjesto jedan po jedan. To čini aplikacije znatno bržim.

Zašto async?

Bez async (sekvencionalno): Sa asyncio.gather() (paralelno): ───────────────────────────────── ───────────────────────────────── Poziv 1: ████████ (2s) Poziv 1: ████████ Poziv 2: ████████ (2s) Poziv 2: ████████ Poziv 3: ████████ (2s) Poziv 3: ████████ ────────────────── Ukupno: 6 sekundi ❌ Ukupno: ~2 sekunde ✅

asyncio.gather() — pokretanje paralelnih zadataka

import asyncio

async def jedan_poziv(temperatura: float) -> str:
    odgovor = await klijent.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": "Sta je AI?"}],
        temperature=temperatura
    )
    return odgovor.choices[0].message.content

async def tri_temperature():
    # Pokrece SVA tri poziva ISTOVREMENO
    rezultati = await asyncio.gather(
        jedan_poziv(0.0),
        jedan_poziv(0.7),
        jedan_poziv(1.5),
    )
    return rezultati  # lista od 3 odgovora
ℹ️ FastAPI je već async: Kada definirate endpoint kao async def, FastAPI automatski rukuje event loop-om. Samo koristite await unutar rute.

🚀 FastAPI — osnove

FastAPI je Python web framework koji automatski generiše dokumentaciju, validira podatke i podržava async.

Tipična struktura FastAPI aplikacije

from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse
from pydantic import BaseModel

app = FastAPI(title="Moja AI Aplikacija")

# Servisiranje statickih fajlova (HTML, CSS, JS)
app.mount("/static", StaticFiles(directory="static"), name="static")

# Pydantic model za validaciju request body
class ZahtjevModel(BaseModel):
    prompt: str
    temperatura: float = 0.7

# GET endpoint — jednostavno citanje
@app.get("/api/info")
async def info():
    return {"verzija": "1.0", "model": "deepseek-chat"}

# POST endpoint — prima JSON body
@app.post("/api/generiraj")
async def generiraj(zahtjev: ZahtjevModel):
    # zahtjev.prompt i zahtjev.temperatura su automatski validirani
    return {"odgovor": "..."}

# Glavna stranica
@app.get("/")
async def root():
    return FileResponse("static/index.html")

Automatska dokumentacija

FastAPI automatski generiše interaktivnu API dokumentaciju. Kada server radi, otvorite u browseru:

  • http://localhost:8001/docs — Swagger UI (interaktivno testiranje)
  • http://localhost:8001/redoc — ReDoc (lijepa dokumentacija)

🛠️ Troubleshooting — česti problemi

❌ UnicodeDecodeError pri pip install

Simptom: 'charmap' codec can't decode byte 0x8d

Uzrok: Dijakritički znakovi (č, ć...) u komentarima requirements.txt

Rješenje: Koristite samo ASCII znakove u komentarima fajla.

❌ error: can't find Rust compiler (tiktoken)

Simptom: Greška pri instalaciji tiktoken-a

Uzrok: tiktoken==0.7.0 nema pre-built wheels za Python 3.13

Rješenje: U requirements.txt promijenite na tiktoken>=0.8.0

❌ ModuleNotFoundError: No module named 'websockets.legacy'

Simptom: Server se ne pokreće, javlja grešku za websockets

Uzrok: Pokrenuta globalna verzija uvicorna umjesto one u venv-u

Rješenje: Koristite python -m uvicorn umjesto samo uvicorn

❌ ModuleNotFoundError: No module named 'fastapi' (ili drugi paket)

Simptom: Python ne pronalazi instalirani paket

Uzrok: Venv nije aktiviran, ili paketi nikada nisu instalirani

Rješenje:

# Provjerite je li venv aktivan (vidite li (venv) u promptu?)
.\venv\Scripts\Activate.ps1

# Instalirajte pakete
pip install -r requirements.txt

# Provjerite instalaciju
pip list

❌ RuntimeError: Ni DeepSeek API ni Ollama nisu dostupni

Uzrok: Nema ni API ključa ni pokretanog Ollama servera

Rješenje A: Dodajte DEEPSEEK_API_KEY=sk-... u .env

Rješenje B: Pokrenite Ollama: docker-compose up ollama

❌ Bijela stranica ili "This site can't be reached"

Uzrok: Server nije pokrenut ili koristite pogrešan URL/port

Rješenje: Pokrenite server i idite na http://localhost:8001 (ne https!)
