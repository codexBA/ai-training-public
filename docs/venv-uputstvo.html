Python Virtual Environment (venv)

Šta je to, zašto je neophodno i kako se pravilno koristi na Windows-u.

1. Šta je to "Virtual Environment"?

Zamislimo da ste vi majstor koji popravlja automobile. Imate ogromnu radionicu u kojoj se nalazi jedan veliki zajednički sanduk sa alatom (to je vaš glavni instalirani Python na računaru).

Danas radite na popravci starog automobila (Projekat A) za koji vam treba specifičan francuski ključ veličine 10. Sutra radite na potpuno novom, modernom automobilu (Projekat B) koji zahtijeva identičan ključ, ali potpuno drugačijeg oblika i veličine 15.

Ako oba ključa bacite u onaj jedan ogromni zajednički sanduk sa alatom — nastaće totalni haos! Svaki put kad posegnete za alatom, izvući ćete pogrešan i nećete moći popraviti auto.

Virtual Environment (venv) rješava taj problem.

Umjesto da sve alate (Python biblioteke) trpate na jedno mjesto, `venv` vam omogućava da za svaki automobil (projekat) kupite mali, odvojeni sandučić sa alatom koji stoji tik pored tog automobila. U njega stavite samo one alate koji vam trebaju za taj specifični auto. Kada završite popravku, taj mali sanduk zatvorite i idete do drugog automobila sa njegovim posebnim sandukom.

2. Da li ga treba koristiti i zašto?

Kratak odgovor: DA! OBAVEZNO! U profesionalnom programiranju, nikada, ali baš nikada ne instaliramo pakete direktno u glavni Python na računaru.

Glavne prednosti korištenja virtuelnih okruženja su:

  • Sprečavanje konflikta verzija ("Dependency Hell"): Vaš stari projekat iz 2023. godine možda koristi biblioteku openai==0.28, dok novi projekat iz ove godine zahtijeva openai==1.50. Nemoguće je imati obje verzije instalirane na istom mjestu. Sa venv-om, oba projekta žive u harmoniji i rade bez problema.
  • Održavanje čistoće sistema: Vaš glavni Python ostaje savršeno čist i neotpakovan, bez stotina bespotrebnih biblioteka koje vam nikada više neće trebati.
  • Lako dijeljenje koda: Kada u vašem venv sandučiću imate samo alate za taj jedan projekat, lako možete popisati koje ste alate koristili (kreirati requirements.txt). Kad taj fajl pošaljete kolegi, on na jedan klik instalira identičan "sandučić" kod sebe i sve mu radi iz prve!

3. Da li se kreira odvojeno za svaki projekat?

Apsolutno DA. Svaki projekat mora imati svoj vlastiti venv folder unutar samog projekta.

Kada radite edukaciju i prelazite sa Dana 1 na Dan 2, vi napuštate jedan "automobil" i odlazite drugom. U folderu za Dan 2 kreirat ćete potpuno novo okruženje i u njega instalirati zavisnosti specifične za taj dan.

Da li venv ide na GitHub?

NIKADA. Folder venv sadrži stotine megabajta koda od samog Pythona i biblioteka. Mi ga nikada ne šaljemo na internet (dodajemo ga u .gitignore fajl). Mi šaljemo samo mali tekstualni fajl requirements.txt (spisak alata). Druga osoba preuzme naš kod, kreira Prazan venv kod sebe i onda pusti Python da skine alate sa spiska.

4. Kako se koristi? (Korak po korak za Windows)

Proces rada sa virtuelnim okruženjem uvijek ima 4 ista koraka:

Korak 1: Kreiranje okruženja (Radi se samo jednom po projektu)

Otvorite terminal (PowerShell) i pozicionirajte se u folder vašeg projekta (npr. cd projekti/dan1). Zatim kucajte:

python -m venv venv

Objašnjenje: Ovo govori Pythonu da pokrene svoj ugrađeni modul za virtual environments (`-m venv`) i da kreira folder koji će se zvati "venv" (posljednja riječ u komandi). U vašem folderu će se sada magično pojaviti folder nazvan venv.

Korak 2: Aktivacija okruženja (Radi se svaki put kad otvorite terminal)

Sada morate reći vašem terminalu: "Hej, od sada pa nadalje, koristi moj odvojeni sandučić sa alatom, a ne onaj glavni!". To se zove aktivacija.

.\venv\Scripts\Activate.ps1

Nakon ovoga, na samom početku linije u terminalu pojavit će se zeleni natpis (venv). To je znak da ste uspješno "ušli" u svoje izolirano okruženje!

Česta greška na Windowsima!

Ako prilikom aktivacije dobijete crvenu grešku "cannot be loaded because running scripts is disabled on this system", to znači da vam Windows zabranjuje pokretanje skripti radi sigurnosti. Upišite ovo jednom u terminal da to dozvolite:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

I zatim ponovite komandu za aktivaciju iz Koraka 2.

Korak 3: Instalacija paketa

Sada kada ste aktivirali venv (vidite onaj zeleni natpis), svaki put kada iskoristite pip install, ti paketi neće ići u vaš cijeli Windows sistem, već isključivo u onaj skriveni folder venv u vašem projektu!

# Primjer pojedinačne instalacije
pip install openai

# Ili još bolje, instalacija svih potrebnih paketa sa spiska
pip install -r requirements.txt

Korak 4: Deaktivacija (Završili ste s radom)

Kada završite rad na tom projektu i želite preći na nešto drugo (ili samo želite izaći iz izoliranog okruženja), jednostavno u terminal ukucajte:

deactivate

Zeleni natpis (venv) će nestati, i ponovo koristite globalni Python vašeg računara.

5. Kompletan Workflow od nule do pokretanja projekta

Evo tačnih koraka koje trebate uraditi svaki put, bez izuzetaka:

📋 Cheat Sheet — Svaki novi projekat (jednom)

# 1. Pozicionirajte se u folder projekta
cd projekti\dan1-tokeni

# 2. Kreirajte virtualno okruženje
python -m venv venv

# 3. Aktivirajte ga
.\venv\Scripts\Activate.ps1

# 4. Instalirajte biblioteke (jednom!)
pip install -r requirements.txt

# 5. Pokrenite server
python -m uvicorn main:app --reload --port 8001

🔄 Svaki sljedeći put (venv već postoji)

# 1. Pozicionirajte se u folder projekta
cd projekti\dan1-tokeni

# 2. Aktivirajte venv
.\venv\Scripts\Activate.ps1

# 3. Pokrenite server (paketi su već instalirani)
python -m uvicorn main:app --reload --port 8001
Zašto python -m uvicorn a ne samo uvicorn?

Ako na vašem računaru postoji i globalno instaliran uvicorn (npr. u AppData\Roaming\Python), komanda uvicorn bez prefiksa može pokrenuti tu globalnu verziju — koja nije u vašem venv-u i može imati drugačije zavisnosti. Korištenjem python -m uvicorn garantujete da se uvijek koristi verzija unutar vašeg virtuelnog okruženja.

6. Brisanje i reset virtualnog okruženja

Ponekad ćete htjeti potpuno obrisati venv i krenuti ispočetka — na primjer, ako instalacija nije prošla kako treba, ili ako želite testirati proces od nule.

Važno: Prije brisanja obavezno deaktivirajte virtualno okruženje. Ne možete obrisati aktivan venv.

Korak 1: Deaktivacija (ako je venv trenutno aktivan)

Ako vidite (venv) na početku terminala, ukucajte:

deactivate

Natpis (venv) nestaje — sad ste van virtualnog okruženja.

Korak 2: Brisanje venv foldera

Na Windows PowerShell-u:

# Potpuno brisanje foldera venv i svega unutar njega
Remove-Item -Recurse -Force .\venv
Alternativa na Linux/Mac: rm -rf venv/

Korak 3: Kreiranje novog čistog venv-a

# Ponovo kreirajte venv
python -m venv venv

# Aktivirajte ga
.\venv\Scripts\Activate.ps1

# Instalirajte pakete ponovo
pip install -r requirements.txt

Gotovo! Imate potpuno novo, čisto virtuelno okruženje.

7. Česti problemi na Windowsu i rješenja

⚠️ Greška: UnicodeDecodeError pri pip install

Simptom: UnicodeDecodeError: 'charmap' codec can't decode byte 0x8d

Uzrok: Komentari u requirements.txt sadrže naša slova (č, ć, š, đ, ž). Windows po defaultu čita fajlove u cp1252 kodiranju koje ne podržava te znakove.

Rješenje: Zamijenite dijakritičke znakove u komentarima fajla requirements.txt:

# Staro (ne radi na Windows):
# uvicorn = ASGI server — pokreće FastAPI aplikaciju

# Novo (radi svuda):
# uvicorn = ASGI server - pokretanje FastAPI aplikacije

⚠️ Greška: tiktoken "No Rust compiler" na Python 3.13

Simptom: error: can't find Rust compiler pri instalaciji paketa

Uzrok: Stara verzija tiktoken==0.7.0 nema pre-kompajlirane binarne fajlove (wheels) za Python 3.13 na Windows-u, pa pokušava kompajlirati izvorni kod u Rustu — za što je potreban instaliran Rust kompajler.

Rješenje: Koristite noviju verziju koja ima gotove wheels:

# U requirements.txt promijenite:
# tiktoken==0.7.0   ← ovo ne radi na Python 3.13
tiktoken>=0.8.0     # ← ovo ima gotove binarne fajlove

Instalirana verzija će biti tiktoken-0.13.0 ili novija — potpuno kompatibilna.

⚠️ Greška: ModuleNotFoundError pri pokretanju uvicorna

Simptom: ModuleNotFoundError: No module named 'websockets.legacy' ili slično

Uzrok: Pokrenuta je globalna verzija uvicorn-a umjesto one iz venv-a. Globalni uvicorn može biti incompatibilan s trenutnim okruženjem.

Rješenje A (preporučeno): Uvijek koristite:

python -m uvicorn main:app --reload --port 8001

Rješenje B: Eksplicitno koristite Python iz venv-a:

.\venv\Scripts\python.exe -m uvicorn main:app --reload --port 8001

⚠️ Greška: venv nije instaliran u venv-u (No module named uvicorn)

Simptom: Nakon aktivacije venv-a, python -m uvicorn javlja "No module named uvicorn"

Uzrok: Biblioteke nikada nisu instalirane unutar venv-a. Vjerojatno ste kreirali venv ali preskočili korak sa pip install.

Rješenje:

# Aktivirajte venv
.\venv\Scripts\Activate.ps1

# Instalirajte biblioteke
pip install -r requirements.txt

# Sada pokrenite
python -m uvicorn main:app --reload --port 8001
