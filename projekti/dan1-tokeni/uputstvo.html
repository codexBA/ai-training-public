Dan 1: Token Vizualizator & Temperature Playground

Dobrodošli na prvi dan vašeg AI putovanja! Danas ćete napraviti nešto što većina ljudi nikada ne vidi — pogledat ćete kako AI zaista vidi tekst. Umjesto da čitate riječi kao mi, AI tekst "siječe" na komade zvane tokeni. Vi ćete danas vidjeti te tokene, obojene u različite boje, i razumjeti zašto je bosanski tekst "skuplji" od engleskog za AI obradu.

Osim toga, eksperimentisaćete s temperaturom — parametrom koji kontroliše koliko je AI "kreativan". Vidjet ćete kako isti AI model može dati potpuno drugačije odgovore samo promjenom jednog broja.

Ovaj vodič je napisan izuzetno detaljno kako biste mogli pratiti korak po korak, čak i ako nikada ranije niste programirali i čak i ako niste bili na predavanju.

Šta ćete naučiti

  • Kako AI modeli vide tekst (tokeni umjesto riječi)
  • Zašto bosanski tekst troši više tokena (i košta više) od engleskog
  • Kako temperatura kontroliše kreativnost AI odgovora
  • Kako izračunati koliko košta jedan API poziv
  • Kako se prebaciti između cloud AI-a (DeepSeek) i lokalnog AI-a (Ollama)
  • Kako pokrenuti web aplikaciju i Jupyter notebook

Šta ćete napraviti

Web aplikaciju s dva dijela:

  • Token Vizualizator — unesete tekst, AI ga razbije na tokene i prikaže obojene blokove. Vidite tačno kako model "čita" vaš tekst.
  • Temperature Playground — unesete pitanje, aplikacija ga pošalje AI-u na 3 različite temperature (0.0, 0.7, 1.5) i prikaže 3 potpuno različita odgovora jedno pored drugog.

Dodatno: 2 CLI programa za terminal i 1 Jupyter notebook za interaktivno učenje.

Zanimljivost: Zašto su tokeni važni?

Svaki put kada koristite ChatGPT, DeepSeek ili bilo koji AI, plaćate po broju tokena. Cijela serija Harry Potter (svih 7 knjiga) ima oko 1.1 milion tokena — čitanje tog teksta kroz GPT-4 bi koštalo oko $2.75. Jedan prosječan email? Oko 270 tokena, ili manje od 0.01 centa. Na ovom treningu ćete potrošiti ukupno manje od $1 na API pozive — jeftinije od jedne kafe.

2b Ollama modeli — šta su, zašto postoje i kako ih koristiti

U vašem .env fajlu postoje dvije naizgled misteriozne varijable: OLLAMA_MODEL_SMALL=llama3.2:1b i OLLAMA_MODEL_MEDIUM=llama3.2:3b. Ova sekcija precizno objašnjava šta su, kako aplikacija bira između njih, da li mogu raditi istovremeno, i kako ih promijeniti.

Šta je Ollama i zašto postoji?

Ollama je program koji vam omogućava da pokrenete AI modele na svom računaru — bez interneta, bez plaćanja, bez slanja podataka negdje drugdje. Zamislite ga kao "Docker za AI modele": jedan server koji može preuzimati, pokretati i upravljati različitim AI modelima.

Na ovom projektu, Ollama je rezervna opcija (fallback) — ako imate DeepSeek API ključ, aplikacija ga koristi. Ako nemate ključ (ili nema interneta), aplikacija se automatski prebaci na Ollamu i koristi AI model koji radi lokalno na vašem računaru.

Analogija: Ollama = "garaža", modeli = "automobili"

Garaža (Ollama) je uvijek na istom mjestu: http://localhost:11434. Modeli (llama3.2:1b, llama3.2:3b...) su kao automobili — možete imati više u garaži, ali koristite onaj koji vam treba u datom trenutku. Promjena modela = "izvaditi drugi auto iz garaže". Garaža se ne mijenja.

Šta su OLLAMA_MODEL_SMALL i OLLAMA_MODEL_MEDIUM?

Ove dvije varijable su samo tekstualne oznake za nazive AI modela. Same po sebi ne rade ništa — vrijednost dobivaju tek kada ih Python kod pročita.

Varijabla u .env Podrazumijevana vrijednost Veličina na disku RAM koji troši Namjena
OLLAMA_MODEL_SMALL llama3.2:1b ~700 MB ~2 GB RAM Brzi odgovori, testiranje, radi na starijim laptopovima
OLLAMA_MODEL_MEDIUM llama3.2:3b ~2 GB ~4 GB RAM Bolji kvalitet odgovora, nešto sporiji, treba 8+ GB RAM

Kako aplikacija koristi ove varijable — pogled u kod

Evo tačnog mjesta u main.py gdje se varijabla čita. Pogledajte funkciju get_llm_client(): 

# Iz main.py — funkcija get_llm_client()

# Prioritet 1: DeepSeek API (cloud)
if deepseek_key and deepseek_key.startswith("sk-"):
    return AsyncOpenAI(api_key=deepseek_key, base_url="https://api.deepseek.com"), "deepseek-chat"

# Prioritet 2: Ollama lokalno (fallback)
ollama_url = os.getenv("OLLAMA_BASE_URL", "http://localhost:11434")
model = os.getenv("OLLAMA_MODEL_SMALL", "llama3.2:1b")
#                  ↑ čita iz .env fajla    ↑ default ako varijabla ne postoji
return AsyncOpenAI(api_key="ollama", base_url=f"{ollama_url}/v1"), model

Vidite: aplikacija uvijek čita OLLAMA_MODEL_SMALL kada koristi Ollamu. OLLAMA_MODEL_MEDIUM je u .env fajlu dostupan za buduće projekte ili za slučaj kada treba prebaciti aplikaciju na bolji model — bez mijenjanja ijedne linije Python koda.

Kada aplikacija koristi koji model?

Tačan redoslijed odlučivanja (prioritet)

  1. DeepSeek API — koristi se ako u .env fajlu postoji DEEPSEEK_API_KEY koji počinje s sk- i nije zakomentarisan. Ovo je cloud model, brz i pametan, ali šalje podatke na server u Kini.
  2. Ollama (SMALL model) — koristi se ako DeepSeek nije dostupan. Aplikacija provjeri da li Ollama radi na localhost:11434, i ako radi, uzme model iz varijable OLLAMA_MODEL_SMALL.
  3. Greška — ako ni DeepSeek ni Ollama nisu dostupni, aplikacija vraća poruku: "Ni DeepSeek API ni Ollama nisu dostupni!"

Mogu li dva Ollama modela raditi istovremeno?

Tehnički: Da. Praktično: Samo ako imate dovoljno RAM-a.

Kada aplikacija pozove model, Ollama ga učita u RAM računara. On tamo ostaje otprilike 5 minuta od zadnjeg zahtjeva. Ako u tom periodu pozovete drugi model, Ollama može oba držati u memoriji — ali to ovisi o RAM-u:

RAM računara Može li istovremeno? Šta se dešava
4 GB ❌ Ne Samo jedan model. Drugi se učita tek kad se prvi istisne iz memorije.
8 GB ⚠️ Teško Moguće za dva SMALL modela, ali računar može postati spor.
16 GB+ ✅ Da SMALL + MEDIUM istovremeno funkcionišu bez problema.

Važno: zahtjevi su sekvencijalni, ne paralelni

Čak i ako su oba modela učitana u RAM, Ollama ih ne obrađuje paralelno. Svaki zahtjev čeka red. Ovo je u redu za edukaciju (jedan student, jedan zahtjev u datom trenutku), ali nije pogodno za produkcijsku primjenu s više korisnika.

Kako preuzeti Ollama modele (mora se uraditi jednom)

Ollama server mora biti pokrenut (Docker kontejner), i model mora biti preuzet prije nego aplikacija pokuša ga koristiti. Preuzimanje se radi jednom — model se čuva u Docker volumenu i preživljava restarte. 

# Korak 1: Pokrenite Ollama kontejner
cd docker/ollama
docker compose up -d

# Korak 2: Preuzmite SMALL model (llama3.2:1b — ~700 MB, traje 2-5 min)
docker exec ai-trening-ollama ollama pull llama3.2:1b

# Korak 3 (opcionalno): Preuzmite MEDIUM model (llama3.2:3b — ~2 GB)
docker exec ai-trening-ollama ollama pull llama3.2:3b

# Provjera — lista preuzetih modela
docker exec ai-trening-ollama ollama list

# Provjera — koji modeli su TRENUTNO učitani u RAM
docker exec ai-trening-ollama ollama ps
# Očekivani output komande: ollama list
NAME             ID              SIZE      MODIFIED
llama3.2:1b      baf6a787fdff    1.3 GB    2 minutes ago
llama3.2:3b      a80c4f17acd5    2.0 GB    1 hour ago

Korištenje Ollame bez koda (Chat direktno iz Terminala)

Jedna od najboljih karakteristika Ollame je to što možete razgovarati s modelom direktno iz Command Prompt-a, PowerShell-a ili terminala, bez pisanja ijedne linije koda.

  • Ako Ollamu pokrećete kroz Docker: 
    docker exec -it ai-trening-ollama ollama run llama3.2:1b
  • Ako imate Ollamu instaliranu lokalno na računaru (van Dockera): 
    ollama run llama3.2:1b

Ovo će pokrenuti interaktivni chat režim u vašem terminalu. Možete upisati pitanje, pritisnuti Enter i model će vam odmah odgovoriti.

Kako izaći iz chata?

Kada želite prekinuti razgovor i zatvoriti chat u terminalu, upišite /bye i pritisnite Enter, ili pritisnite Ctrl + D.

Kako prebaciti aplikaciju na drugi model

Ne trebate mijenjati ni jednu liniju Python koda. Samo promijenite .env fajl i restartujte server: 

# U .env fajlu promijenite:
OLLAMA_MODEL_SMALL=llama3.2:3b   # koristite 3B umjesto 1B

# Ili promijenite na potpuno drugačiji model (koji ste prethodno preuzeli):
OLLAMA_MODEL_SMALL=mistral:7b

Nakon promjene:

  1. Sačuvajte .env fajl
  2. Zaustavite server: Ctrl+C u terminalu gdje radi uvicorn
  3. Pokrenite server ponovo: python -m uvicorn main:app --reload --port 8001
  4. U log porukama vidjet ćete koji model se sad koristi

Česta greška: .env promijenjen, server nije restartovan

.env fajl se čita samo pri pokretanju aplikacije. Ako promijenite fajl dok server radi — promjena NEĆE stupiti na snagu. Uvijek restartujte server nakon promjene .env!

Provjera koji model aplikacija koristi (u terminalu)

Kada pokrenete server, pogledajte prve linije ispisa u terminalu. Tamo ćete vidjeti koji AI provider je aktivan: 

# Ako koristi DeepSeek:
INFO:     LLM provider: DeepSeek (deepseek-chat)

# Ako koristi Ollamu:
INFO:     LLM provider: Ollama (llama3.2:1b)

Ako ne vidite ove poruke, provjerite main.py — možda aplikacija nema explicit logging na startu. U svakom slučaju, možete provjeriti koji model je aktivan pozivom endpointa: http://localhost:8001/api/model-info

Izazov 1: Engleski vs Bosanski

Probajte unijeti ove parove i zapišite koliko tokena ima svaki:

Engleski tekst Bosanski tekst Vaš rezultat (EN) Vaš rezultat (BS)
"Hello world" "Pozdrav svijete" ___ ___
"Artificial intelligence" "Vještačka inteligencija" ___ ___
"The weather is nice today" "Vrijeme je danas lijepo" ___ ___

Zapažanje: Bosanski tekst gotovo uvijek ima više tokena od engleskog za istu informaciju. Razlog: tokenizator (BPE algoritam) je treniran pretežno na engleskom tekstu, pa engleske riječi efikasnije kodira. Bosanski tekst košta ~30-50% više tokena. Ovo je važno za budžetiranje API troškova!

Izazov 2: Kod vs Proza

Probajte tokenizirati programski kod:

  • print("Hello, World!")
  • SELECT * FROM users WHERE id = 1
  • def calculate_tax(income, rate):

Zapažanje: Ključne riječi programskih jezika (print, SELECT, FROM, def) su obično po 1 token jer su izuzetno česte u trening podacima. AI je treniran na ogromnoj količini koda!

Izazov 3: Brojevi, JMBG i specijalni znakovi

Probajte ove specijalne slučajeve:

  • JMBG: 0101990170001 — Kako AI "vidi" JMBG?
  • erkan@example.com — Koliko tokena ima email adresa?
  • 2024-12-25T14:30:00Z — A datum u ISO formatu?

Zapažanje: Brojevi se obično dijele u grupe od 1-3 cifre po tokenu. Specijalni znakovi (@, :, /, .) su obično po 1 token svaki. JMBG od 13 cifara može biti 5-6 tokena — relativno "skupo" za tako malo informacija.

Izazov 4: Temperatura za različite zadatke

Testirajte ove promptove i posmatrajte kako temperatura utiče na svaki:

Prompt Očekivanje za t=0.0 Očekivanje za t=1.5
"Koliko je 15 + 27?" Tačan odgovor: 42 Možda tačno, možda ne — nepouzdano za matematiku!
"Napiši kratku priču o robotu u Sarajevu" Suhoparno, predvidljivo Kreativno, originalno, ponekad čudno
"Napiši Python funkciju za zbrajanje dva broja" Standardno, korektno rješenje Možda kreativan pristup, ali potencijalno neispravan
"Objasni šta je GDPR u jednoj rečenici" Precizna, kratka definicija Možda poetična, ali manje precizna

Zaključak: Za faktičke zadatke (matematika, kod, ekstrakcija podataka) → koristite nisku temperaturu (0.0–0.3). Za kreativne zadatke (priče, marketing, brainstorming) → koristite višu temperaturu (0.7–1.0). Temperaturu iznad 1.0 izbjegavajte u produkciji.

Izazov 5: Eksperiment ponovljivosti

Pošaljite isti prompt 3 puta na temperaturi 0.0. Da li dobijate isti odgovor svaki put? Sada uradite isto na temperaturi 1.0. Koliko se odgovori razlikuju?

Lekcija: Temperatura 0.0 = deterministički (isti input = isti output). Ovo je ključno za aplikacije gdje trebate ponovljive rezultate (npr. ekstrakcija podataka iz faktura).

Šta smo naučili — Kontrolna lista

Ako možete staviti kvačicu pored svakog od ovih, Dan 1 je uspješan:

  • Tokeni nisu riječi — to su podriječi koje AI koristi kao osnovne jedinice teksta
  • Bosanski tekst košta ~30-50% više tokena od engleskog za istu informaciju
  • Temperatura 0.0 = isti odgovor svaki put (deterministički), 1.5 = kreativni haos
  • Za faktičke zadatke koristite nisku temperaturu, za kreativne višu
  • Cloud AI (DeepSeek) = brži i pametniji, Lokalni AI (Ollama) = privatan i besplatan
  • API ključevi se čuvaju u .env fajlu, nikada u kodu
  • Znam pokrenuti web server, otvoriti aplikaciju u browseru i koristiti CLI programe
  • Znam otvoriti i pokrenuti Jupyter notebook

Rješavanje problema (Troubleshooting)

"Port 8001 already in use"

Aplikacija je već pokrenuta u drugom terminalu. Zatvorite stare terminale ili pritisnite Ctrl+C u terminalu gdje server radi. Ako i dalje ne radi, na Windows-u pokrenite: 

netstat -ano | findstr :8001

Nađite PID broj (zadnja kolona) i ugasite proces:

taskkill /PID BROJ /F

"RuntimeError: Ni DeepSeek API ni Ollama nisu dostupni"

Niste unijeli API ključ u .env fajl I lokalni Ollama nije pokrenut. Rješenje: ili dodajte pravi ključ, ili pokrenite Ollama server (docker-compose -f ../../docker-compose.yml up ollama).

"ModuleNotFoundError: No module named 'tiktoken'"

Virtualno okruženje (venv) nije aktivno. Aktivirajte ga: .\venv\Scripts\Activate.ps1. Provjerite da vidite (venv) na početku linije u terminalu.

Bijela stranica u browseru / "This site can't be reached"

Server nije pokrenut ili koristite pogrešan URL. Provjerite da je server aktivan u terminalu i da idete na http://localhost:8001 (ne na https, ne na drugi port).

Odgovor na temperaturu traje predugo (30+ sekundi)

Ako koristite Ollama, lokalni model može biti spor na CPU-u — to je normalno. DeepSeek cloud odgovara za 1-3 sekunde. Ako želite brže odgovore, prebacite se na DeepSeek (dodajte API ključ u .env).

"Docker daemon is not running"

Docker Desktop nije pokrenut. Otvorite Docker Desktop aplikaciju, sačekajte da se pokrene (zelena ikona u system tray-u), i probajte ponovo.

Token vizualizator ne prikazuje ništa

Vjerovatno ste ostavili polje prazno ili unijeli samo razmake. Unesite barem jednu riječ i kliknite dugme ponovo.

Notebook ćelija javlja grešku pri pokretanju

Najčešći razlog: niste odabrali pravi Kernel. Kliknite "Select Kernel" u gornjem desnom uglu VS Code-a i odaberite venv. Ako ne vidite venv, pokrenite terminalu: python -m ipykernel install --user --name=venv.

Bonus izazovi za one koji žele više

Izazov A: Pronađite najdužu bosansku riječ koja je 1 token

Koristite Token Vizualizator da testirate razne riječi. Možete li pronaći bosansku riječ od 6+ slova koja je samo 1 token? (Hint: pokušajte s riječima koje se koriste i u engleskom, poput tehničkih termina.)

Izazov B: Izračunajte cijenu obrade svih emailova

Zamislite da vaša institucija prima 50 emailova dnevno, svaki prosječno 200 riječi. Koliko bi koštalo da AI pročita (tokenizira) sve emailove za mjesec dana koristeći DeepSeek? (Hint: 200 riječi ≈ 270 tokena, DeepSeek košta $0.14/1M tokena)

Izazov C: Temperatura za SQL

U Temperature Playground, unesite: "Napiši SQL upit koji vraća sve korisnike starije od 30 godina". Koja temperatura daje najispravniji SQL? Probajte 0.0, 0.3, 0.7, i 1.5. Zapišite rezultate.

Izazov D: Emoji tokeni

Koliko tokena zauzima jedan emoji? Probajte: 😀, 🇧🇦, 👨‍💻. Rezultat vas može iznenaditi — neki emoji su čak 3-4 tokena!
