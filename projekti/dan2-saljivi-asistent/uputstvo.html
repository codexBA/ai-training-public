Teorijski uvod — Zašto ovaj projekat i šta gradimo?

Dobrodošli u prvi mini-projekat za Dan 2! U ovom projektu gradimo Šaljivog Asistenta — jednostavan chatbot koji može mijenjati svoju "ličnost" i odgovarati kao mrgodni djed, pirat, romantični pjesnik, sarkastična IT podrška ili sportski komentator.

Zašto radimo ovaj projekat i čemu služi?

Cilj ovog projekta je savladavanje samih osnova komunikacije sa modernim Large Language Modelima (LLM) kroz kod, odnosno učenje kako radi Chat API. Prije nego što počnemo praviti kompleksne agente koji pamte duge razgovore i barataju bazama podataka, moramo razumjeti osnovne gradivne blokove. Ovaj projekat služi da demistificira magiju iza ChatGPT-a i pokaže vam da se sve svodi na slanje jednostavnih tekstualnih instrukcija u strukturiranom JSON formatu na server.

Šta želimo demonstrirati?

  • Moć System Prompt-a: Pokazaćemo kako samo jedna rečenica skrivene instrukcije (system prompt) može iz temelja promijeniti vokabular, ton i ponašanje modela, a da pritom koristimo potpuno isti jezički model u pozadini.
  • Parametar Temperature: Demonstriraćemo kako matematički parametar (temperatura od 0.0 do 1.5) upravlja time da li će model biti strogo precizan i predvidiv (0.0), ili izuzetno kreativan i pomalo haotičan (1.5).
  • Osnove backend posredništva: Vidjećemo kako napraviti jednostavan web server u Pythonu (FastAPI) koji služi kao posrednik između našeg korisničkog interfejsa (frontenda) i LLM provajdera (DeepSeek ili Ollama).

Zašto koristimo "stateless" (besstanični) API i zašto nema historije?

Ključna specifičnost ovog projekta u odnosu na naprednije chatbote je ta što ovdje nema historije razgovora. Svaki put kada pošaljete poruku, naš backend šalje samo dvije stvari modelu: System prompt (uloga persone) i Trenutnu poruku korisnika.

Ovo radimo namjerno kako bismo naučili koncept stateless API-ja (API-ja bez stanja). Važno je shvatiti da jezički modeli (LLM) prirodno nemaju memoriju. Oni ne znaju šta ste ih pitali prije 5 sekundi. Da bi chatbot "pamtio" razgovor, mi kao programeri moramo pri svakom novom pitanju iznova poslati modelu cjelokupnu historiju dosadašnjeg razgovora. Ostavljajući ovaj projekat "stateless", jasno demonstriramo tu činjenicu: svaki zahtjev prema modelu je potpuno neovisan događaj izbrisan od svake prošlosti. Razumijevanje ovoga sada je ključno za kasnije optimiziranje potrošnje tokena i pamćenja u većim projektima!

Šta ćemo naučiti?

  • Chat API — kako funkcioniše messages array s rolama system i user
  • System prompt — kako jedna instrukcija potpuno mijenja ponašanje modela
  • Temperatura — kako brojčana vrijednost (0.0–1.5) utiče na kreativnost odgovora
  • Stateless API — zašto model ne pamti prethodne poruke
  • FastAPI endpointe — GET i POST rute, automatska dokumentacija na /docs
  • Frontend fetch — kako browser komunicira s backendom bez slanja historije

Server sluša na http://localhost:8017. Frontend i curl pozivaju ove URL adrese.

GET /api/persone

Svrha: Vraća listu dostupnih persona za frontend dugmad.

curl http://localhost:8017/api/persone

Primjer odgovora:

{
  "dedo": {"ime": "Mrgodni Djed", "opis": "Mrgodni dedo koji na sve gleda skeptično"},
  "pirat": {"ime": "Jadranski Pirat", "opis": "Pirat s Jadranskog mora koji priča o pustolovinama"},
  ...
}

POST /api/chat

Svrha: Šalje poruku odabranoj personi; vraća LLM odgovor.

curl -X POST http://localhost:8017/api/chat ^
  -H "Content-Type: application/json" ^
  -d "{\"poruka\": \"Šta misliš o programiranju?\", \"persona\": \"dedo\", \"temperatura\": 0.7}"

Primjer odgovora:

{
  "odgovor": "*uzdiše* Programiranje... u moje vrijeme...",
  "persona": "dedo",
  "persona_ime": "Mrgodni Djed",
  "model": "deepseek-chat"
}

Korak 6 — Pokretanje i testiranje

Sada kada imate gotov kod, potrebno je da pokrenete projekat.

1. Aktivacija i instalacija

U terminalu pozicionirajte se u folder projekta i kreirajte virtualno okruženje:

python -m venv venv
venv\Scripts\activate  # Za Windows
pip install -r requirements.txt

2. Postavljanje API ključa

Kopirajte .env.example u .env i unesite vaš DeepSeek API ključ.

3. Pokretanje

uvicorn main:app --reload --port 8017

Otvorite http://localhost:8017 u browseru.

Eksperimenti (45 min)

Sada kada aplikacija radi, vrijeme je za eksperimentisanje — ovdje učite zašto stvari rade kako rade.

Eksperiment 1

Temperature slider

Pošaljite potpuno isto pitanje tri puta s različitim temperaturama:

  1. Temperatura 0.0 — "Reci mi vic" → zapišite odgovor
  2. Temperatura 0.7 — isto pitanje → uporedite
  3. Temperatura 1.5 — isto pitanje → uporedite
  • Na 0.0 odgovor je gotovo identičan svaki put
  • Na 0.7 ima malo varijacije
  • Na 1.5 odgovori su nepredvidivi
Zaključak: 0.0–0.3 za činjenice. 0.7–1.0 za kreativne odgovore. Iznad 1.2 rijetko u produkciji.
Eksperiment 2

Kreirajte vlastitu personu

Dodajte 6. personu u PERSONE rječnik. Ideje: kuhar, detektiv, baka, rapper.

    "kuhar": {
        "ime": "Bosanski Kuhar",
        "opis": "Kuhar koji sve povezuje s hranom",
        "prompt": (
            "Ti si bosanski kuhar koji na svako pitanje odgovara kroz prizmu kuhanja i hrane. "
            "Koristiš kulinarsku terminologiju i metafore iz kuhinje. "
            "Svaki savjet završavaš s preporukom nekog bosanskog jela. "
            "Odgovaraj na bosanskom, hrvatskom ili srpskom jeziku. Budi topao i srdačan."
        )
    }

Sačuvajte → server se restartuje (--reload) → osvježite browser.

Eksperiment 3

Šta se desi BEZ system prompta?

Zakomentirajte system poruku u main.py:

messages=[
    # {"role": "system", "content": persona_data["prompt"]},
    {"role": "user", "content": req.poruka},
],
  • Bez system prompta: model odgovara generički
  • Sa system promptom: model ima ličnost

Vratite system prompt nakon testa!

Eksperiment 4

Dokaz da nema historije

  1. Pošaljite: "Moje ime je Ahmet."
  2. Pošaljite: "Kako se zovem?"

Model neće znati vaše ime — svaki poziv šalje samo system + user, bez prethodnog konteksta.

U dan2-film-chat frontend šalje cijelu historiju u svakom pozivu.
