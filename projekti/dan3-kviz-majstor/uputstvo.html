Uvod — Šta ćemo napraviti?

Kviz aplikacija koja čita pitanja iz JSON fajla (kviz.json), nasumično ih prikazuje studentu, a zatim koristi AI da ocijeni slobodne tekstualne odgovore. AI vraća ocjenu kao strukturirani JSON umjesto slobodnog teksta — to nam omogućava da program prikaže bodove, komentar i tačan odgovor bez parsiranja teksta.

Ključni koncept — Strukturirani output: Umjesto da model vrati slobodan tekst ("Bravo, to je tačno..."), mi zahtijevamo da vrati JSON s tačno definiranim poljima. To omogućava programskoj obradi odgovora (bodovi, rezultat, itd.).

Šta ćemo naučiti?

  • Fallback pattern — što raditi kad AI ne poštuje JSON format

API dokumentacija

API (Application Programming Interface) je skup URL adresa koje frontend (ili curl) poziva da dobije podatke. Naš server sluša na http://localhost:8018.

Metoda Putanja Funkcija u kodu Request body Response
GET / root() — HTML stranica (index.html)
GET /api/pitanje daj_pitanje() — JSON s pitanjem (bez tačnog odgovora)
GET /api/kategorije get_kategorije() — JSON niz kategorija
POST /api/provjeri provjeri() {pitanje_id, odgovor_studenta} OcjenaResponse JSON

GET /api/pitanje

Svrha: Dohvata jedno nasumično pitanje i pamti ga na serveru.

Primjer curl (PowerShell):

curl http://localhost:8018/api/pitanje

Primjer odgovora:

{
  "id": 3,
  "pitanje": "Na kojoj rijeci se nalazi Stari Most u Mostaru?",
  "kategorija": "geografija",
  "tezina": 1
}
Primijetite: nema polja tacan_odgovor — namjerno!

GET /api/kategorije

Svrha: Vraća listu svih jedinstvenih kategorija iz kviz.json.

curl http://localhost:8018/api/kategorije
["zabava", "kultura", "geografija", "tech", "hrana"]

Redoslijed može varirati jer koristimo set.

POST /api/provjeri

Svrha: Šalje studentski odgovor; AI ocjenjuje i vraća strukturirani JSON.

Sekvenca: Prvo GET /api/pitanje → zapamtite id → zatim POST s tim id-jem.

curl -X POST http://localhost:8018/api/provjeri ^
  -H "Content-Type: application/json" ^
  -d "{\"pitanje_id\": 3, \"odgovor_studenta\": \"Neretva\"}"

Primjer odgovora:

{
  "tacno": true,
  "bodovi": 10,
  "komentar": "Odličan odgovor! Neretva je rijeka na kojoj stoji Stari Most.",
  "tacan_odgovor": "Neretva"
}

Swagger UI — /docs

FastAPI automatski generiše interaktivnu dokumentaciju. Otvorite u browseru:

http://localhost:8018/docs

  1. Pronađite POST /api/provjeri
  2. Kliknite Try it out
  3. Unesite JSON: {"pitanje_id": 1, "odgovor_studenta": "Bosna"}
  4. Kliknite Execute — vidite odgovor ispod
Prije POST /api/provjeri morate pozvati GET /api/pitanje s istim id-jem u istoj sesiji servera — inače dobijate "Pitanje nije pronađeno".

Korak 7 — Pokretanje i testiranje

Ovaj projekat koristi istu strukturu kao prethodni.

1. Instalacija i API ključ

python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

Postavite .env fajl sa vašim API ključem.

2. Pokretanje

uvicorn main:app --reload --port 8018

Otvorite http://localhost:8018 u browseru.

Sačuvajte → restartujte server → testirajte.

Eksperiment 2

Promijenite scoring shemu (0–100)

class OcjenaResponse(BaseModel):
    bodovi: int = Field(ge=0, le=100)  # Promjena!

Ažurirajte system prompt: "Bodovi: 100=savršen..."

Bonus: Vratite Pydantic na 0–10 ali ostavite prompt na 0–100 — šta se desi kad model vrati 85?

Eksperiment 3

Filter po kategoriji

@app.get("/api/pitanje")
async def daj_pitanje(kategorija: str = None):
    if kategorija:
        filtrirano = [p for p in PITANJA if p["kategorija"] == kategorija]
        pitanje = random.choice(filtrirano) if filtrirano else random.choice(PITANJA)
    else:
        pitanje = random.choice(PITANJA)
    # ... ostatak isti ...

Test: http://localhost:8018/api/pitanje?kategorija=tech

Eksperiment 4

Testirajte s Ollama

Zakomentarišite DEEPSEEK_API_KEY u .env, restartujte server. Primijetite češći fallback na string matching — manji modeli lošije poštuju JSON format.

Eksperiment 5

Povežite frontend s GET /api/kategorije

Dodajte dropdown u index.html koji poziva fetch('/api/kategorije') i šalje odabrani filter na /api/pitanje?kategorija=... (nakon Eksperimenta 3).
